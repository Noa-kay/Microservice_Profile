# MyProject – .NET 7+ Web API

A clean, decoupled N-Layer Web API built with .NET 7+.

---

## Project Structure

```
MyProject/
├── Controllers/        # HTTP endpoints – receives requests, returns responses
├── BLL/                # Business Logic Layer – services and core logic
├── Data/
│   ├── Models/         # Database entities (EF Core)
│   ├── Context/        # DbContext definition
│   └── Repositories/  # Data access queries (optional)
├── DTOs/               # Data Transfer Objects – API contracts
└── Migrations/         # Auto-generated by EF Core
```

---

## Getting Started

```bash
dotnet new webapi -n MyProject
cd MyProject
mkdir Data Data/Models Data/Context Data/Repositories BLL DTOs
```

---

## Data Flow

```
Database → Entity → Repository → Service (BLL) → DTO → Controller → Client
```

---

## Team Roles

| Role | Layer | Responsibility |
|---|---|---|
| Data Developer | `Data/` | DbContext, Entities, Migrations |
| BLL Developer | `BLL/` | Interfaces & Service implementations |
| DTOs & Mapping Developer | `DTOs/` | DTOs + AutoMapper configuration |
| Controllers Developer | `Controllers/` | Endpoints, DI registration, Swagger |

### Recommended Workflow

1. **Data + DTOs developers start first** – define models and contracts
2. **BLL developer** builds services based on the data layer
3. **Controllers developer** wires everything to the API
4. Each developer works on a **separate Git branch** and merges after review

## Service Registration

In `Program.cs`, register your services for Dependency Injection:

```csharp
builder.Services.AddScoped<IUserService, UserService>();
```

## Code Examples

### DTO

```csharp
namespace MyProject.DTOs
{
    public class UserDto
    {
        public int Id { get; set; }
        public string Username { get; set; } = string.Empty;
    }
}
```

### Service Interface & Implementation (BLL)

```csharp
namespace MyProject.BLL
{
    public interface IUserService
    {
        IEnumerable<UserDto> GetAllUsers();
    }

    public class UserService : IUserService
    {
        public IEnumerable<UserDto> GetAllUsers()
        {
            // Fetch from Data layer and map to DTOs
            return new List<UserDto>();
        }
    }
}
```

## Pro Tips

- **Interfaces** – Always define an interface per service to enable easy unit testing and swapping implementations.
- **AutoMapper** – Automates mapping between Entities and DTOs, eliminating repetitive manual mapping code.
- **FluentValidation** – Keeps validation logic clean and decoupled from DTOs.
- **Global Usings** – Use a `GlobalUsings.cs` file to avoid repetitive `using` statements across files.
